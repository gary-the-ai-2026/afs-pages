Production Output

CAC Attribution
Model

Built a channel-weighted, GA4-validated customer acquisition cost model across both brands. Previously this data didn't exist anywhere in the business.

What It Does

• Calculates true CAC per customer segment across two brands
• Breaks down by B2C Online, B2C Home Gym, B2B Commercial, B2B Enterprise
• Maps revenue per customer and margin per segment
• Validates paid attribution rates against GA4 first-touch data
• Handles dark traffic redistribution (customers with no tracking data)
• Classifies customer archetypes from phone transcript JTBD matching

Five-Step Pipeline

01 Spend allocation from Growth Matrix into 3 tiers (Paid / Digital / Overhead)
02 Customer classification via NetSuite tags, department codes, and spend thresholds
03 Channel attribution from Shopify landing pages + HubSpot first-touch URLs
04 GA4 validation of paid rates + dark traffic redistribution
05 Cost allocation by channel exposure per archetype

Result: For the first time, we can see exactly which customer segments are most profitable, which channels deliver them, and where we're overspending. This model now drives budget allocation decisions across both brands.

Governance & Trust

Built Responsibly,
Not Just Fast

An agent that can send emails, post to Asana, and action tasks needs guardrails. I built a 4-tier trust framework from day one.

Trust Tiers

Tier 1 Autonomous
Read, organise, learn
Tier 2 Draft & Review
Draft anything, Josh approves
Tier 3 Action with Audit
Can act, must leave trail
Tier 4 Never Autonomous
Financial, external, irreversible

Key Guardrails

• Draft Approval Protocol - Nothing goes external without explicit sign-off
• NetSuite Read-Only - ERP is never written to without consent
• Shared Platform Rule - Never replies unless explicitly @tagged
• Audit Trail - Every Asana action gets a comment explaining what was done and why
• Duplicate Check - Heartbeats verify no existing response before acting
• CC Protocol - Josh is CC'd on every outbound email automatically
• Feedback Memory - Corrections persist across sessions so mistakes happen once
• Model Governance - Opus for judgment, Sonnet for routine, documented rationale
